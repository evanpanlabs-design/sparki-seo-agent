### 交互设计

| 按键 | 动作 |
|------|------|
| `Enter` | 提交当前输入 |
| `↑ / ↓` | 浏览历史命令 |
| `/` | 进入命令模式 (`/add`, `/status`, `/cancel`) |
| `Tab` | 自动补全 |
| `Ctrl+C` | 取消当前任务 |
| `Ctrl+L` | 清屏 |
| `1-9` | 快速切换任务焦点 |

### 状态展示
- 每个任务用一区块展示，包含 task_id、平台图标、进度条、阶段节点、实时日志
- 运行中任务用 `●` 前缀+动态进度条
- 完成用绿色 `✓`，失败用红色 `✗`
- AI 对话区类似 Claude Code 的输入输出样式

## Sparki 设计风格参考

从 sparki.io 页面提取的设计规范：

### 配色方案
| 元素 | 颜色值 | 用途 |
|------|--------|------|
| 主色调 | `#5E33BF` | 按钮背景、强调色 |
| 主色悬停 | `#6b40cc` | 按钮 hover 状态 |
| 深色文字 | `#111111` | 主要文字 |
| 浅色背景 | `#f0f0eb` | 卡片/区块背景 |
| 白色 | `#ffffff` | 按钮背景、卡片 |
| 次要文字 | `#646363` | 次要文字、边框 |

### 字体
- 主字体: `Source Sans Pro` (300, 400, 600, 700)
- 字重轻到重: 200 italic → 900

### UI 元素规范
- **按钮圆角**: `4px`
- **按钮边框**: `1px solid` 对应色
- **按钮阴影**: `0 0 0 1px transparent`（保持简洁）
- **悬停动画**: `transition: background-color 0.2s ease-in`
- **图标按钮**: `border-radius: 50%`
- **卡片**: 简洁风格，无厚重阴影

### 设计理念
- **简洁克制**: 没有夸张的阴影或渐变
- **紫色为主**: `#5E33BF` 作为品牌色，替代我之前设想的 Indigo
- **功能导向**: 每个元素服务于特定目的
- **hover 反馈**: 紫色按钮 hover 时变为白底紫字（反转）

### 6个核心组件

#### 1. AI 对话框 (AI Chat Panel)
- 位于左上区域
- 输入框 + 发送按钮（紫色主色调，4px圆角）
- 历史对话可滚动
- 快捷命令提示：`/add [url]`, `/status`, `/cancel [id]`

#### 2. 日志框 (Log Panel)
- 实时流式日志输出
- 按任务分组，可折叠/展开
- 日志级别着色：INFO(紫)、WARN(黄)、ERROR(红)
- 支持搜索/筛选

#### 3. 视频播放框 (Video Player)
- 显示视频缩略图 + 元数据
- 播放/暂停控制（圆形图标按钮）
- 显示：作者、时长、清晰度、点赞/播放数
- 点击可全屏预览

#### 4. Blog 预览 & 质检框 (Blog & QC Preview)
- Markdown 渲染的博客正文
- 质检结果评分 + 问题列表
- 一键重新生成/发布（紫色按钮）
- 可编辑 markdown

#### 5. 项目选择框 (Library Panel)
- 左侧边栏，项目列表
- 每个项目显示：任务总数、运行中、完成、失败
- 筛选标签：全部 / 运行中 / 完成 / 失败
- 新建项目按钮（白底紫字边框）

#### 6. 进度条 (Progress Bar)
- 底部全局进度条
- 当前任务名称 + 百分比
- 阶段指示器：[下载]→[元数据]→[分析]→[文章]→[质检]→[发布]

| 背景 | `#f0f0eb` (Sparki米白) | `#111111` (深黑) |

| 边框 | `#e9eaed` | `#333333` |
| 主色调 | `#5E33BF` (Sparki紫) | `#7c4dff` |

| 警告 | `#F59E0B` | `#FBBF24` |

| 文字主 | `#111111` | `#ffffff` |
| 文字次 | `#646363` | `#9d9d9c` |

#### 圆角与阴影
- 按钮圆角: `4px` (遵循Sparki)
- 卡片圆角: `12px`
- 输入框圆角: `4px`
- 卡片阴影: 极简，仅 `0 0 0 1px transparent` 保持层次感
- hover 阴影: `0 0 0 1px #5E33BF` 边框高亮

#### 字体
- 主字体: `Inter`, `-apple-system`, `BlinkMacSystemFont`, sans-serif
- 代码/日志: `JetBrains Mono`, `Consolas`, monospace

#### 间距系统
- 基础单位: `4px`
- 卡片内边距: `24px`
- 组件间距: `16px`
- 区块间距: `32px`

### 技术实现

#### TUI 技术栈
- `Textual` (Python TUI 框架，类似 ClaudeCode 风格)
- 或者 `curses` + `blessed` 手动实现
- 支持 ANSI 颜色和动画

#### GUI 技术栈（两选一）
1. **Web 版**: `Streamlit` / `Gradio` + 自定义 CSS
2. **桌面版**: `PyQt6` / `Electron` (HTML + Node.js)

### 交互模式

| 动作 | 触发 |
|------|------|
| 添加视频 | AI对话框输入URL 或 `/add [url]` |
| 查看任务详情 | 点击任务列表项 |
| 取消任务 | 右键菜单 / `Ctrl+C` |
| 重新生成文章 | Blog预览区点击"重新生成" |
| 发布到 Contentful | Blog预览区点击"发布" |
| 切换主题 | 右上角月亮/太阳图标 |
| 筛选任务 | 项目面板的标签筛选 |

### 按钮风格（参考 Sparki）
- **主按钮**: 紫色背景 `#5E33BF`，白色文字，hover时背景变白、文字变紫（反转效果）
- **次要按钮**: 白色背景，紫色边框，hover时背景变浅紫
- **图标按钮**: 圆形 `border-radius: 50%`，透明背景
- **过渡动画**: `transition: background-color 0.2s ease-in`

## 对比总结

| 维度 | TUI | GUI |
|------|-----|-----|
| 部署难度 | 低（终端即可） | 中（需安装界面库） |
| 学习成本 | 高（需记忆快捷键） | 低（图形化操作） |
| 信息密度 | 高 | 中 |
| 远程友好 | ✓（SSH即可） | 需要GUI转发 |
| 多任务并行展示 | 优秀 | 良好 |
| 视觉呈现 | 受终端限制 | 更丰富 |
| 适用场景 | 开发者/键盘党 | 普通用户 |
